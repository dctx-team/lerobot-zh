# LIBERO

**LIBERO** 是一个旨在研究**终身机器人学习**的基准测试。其理念是机器人不会仅在工厂进行一次预训练，它们需要随着时间的推移与人类用户一起不断学习和适应。这种持续适应被称为**决策中的终身学习 (LLDM)**，它是构建真正个性化助手机器人的关键一步。

- 📄 [LIBERO 论文](https://arxiv.org/abs/2306.03310)
- 💻 [原始 LIBERO repo](https://github.com/Lifelong-Robot-Learning/LIBERO)

为了在这一挑战上取得进展，LIBERO 提供了一组专注于**知识迁移**的标准化任务：机器人能够多好地将已经学到的知识应用于新情况。通过在 LIBERO 上进行评估，可以公平地比较不同的算法，研究人员可以在彼此的工作基础上进行构建。

LIBERO 包括**五个任务套件**：

- **LIBERO-Spatial (`libero_spatial`)** – 需要空间关系推理的任务。
- **LIBERO-Object (`libero_object`)** – 以操作不同对象为中心的任务。
- **LIBERO-Goal (`libero_goal`)** – 目标条件任务，机器人必须适应变化的目标。
- **LIBERO-90 (`libero_90`)** – 来自 LIBERO-100 集合的 90 个短视野任务。
- **LIBERO-Long (`libero_10`)** – 来自 LIBERO-100 集合的 10 个长视野任务。

这些套件总共涵盖了**130 个任务**，从简单的对象操作到复杂的多步骤场景。LIBERO 旨在随着时间的推移而增长，并作为一个共享基准，社区可以在其中测试和改进终身学习算法。

![LIBERO 基准测试概览](https://libero-project.github.io/assets/img/libero/fig1.png)

## 使用 LIBERO 进行评估

在 **LeRobot**，我们将 [LIBERO](https://github.com/Lifelong-Robot-Learning/LIBERO) 移植到我们的框架中，主要用于**评估 [SmolVLA](https://huggingface.co/docs/lerobot/en/smolvla)**，我们的轻量级 Vision-Language-Action 模型。

LIBERO 现在是我们**支持多评估的仿真**的一部分，这意味着你可以使用一个标志在**单个任务套件**或同时**跨多个套件**对你的策略进行基准测试。

要安装 LIBERO，请在遵循 LeRobot 官方说明后，只需执行：
`pip install -e ".[libero]"`

### 单套件评估

在一个 LIBERO 套件上评估策略：

```bash
lerobot-eval \
  --policy.path="your-policy-id" \
  --env.type=libero \
  --env.task=libero_object \
  --eval.batch_size=2 \
  --eval.n_episodes=3
```

- `--env.task` 选择套件（`libero_object`、`libero_spatial` 等）。
- `--eval.batch_size` 控制并行运行多少个环境。
- `--eval.n_episodes` 设置总共运行多少个 episode。

---

### 多套件评估

一次性在多个套件上对策略进行基准测试：

```bash
lerobot-eval \
  --policy.path="your-policy-id" \
  --env.type=libero \
  --env.task=libero_object,libero_spatial \
  --eval.batch_size=1 \
  --eval.n_episodes=2
```

- 将逗号分隔的列表传递给 `--env.task` 以进行多套件评估。

### 策略输入和输出

通过 LeRobot 使用 LIBERO 时，策略通过**观察**和**动作**与环境交互：

- **观察**
  - `observation.state` – 本体感受特征（agent 状态）。
  - `observation.images.image` – 主相机视图（`agentview_image`）。
  - `observation.images.image2` – 手腕相机视图（`robot0_eye_in_hand_image`）。

  ⚠️ **注意：** LeRobot 对任何多模态视觉特征强制使用 `.images.*` 前缀。始终确保你的策略配置 `input_features` 使用相同的命名键，并且你的数据集元数据键在评估期间遵循此约定。
  如果你的数据包含不同的键，你必须重命名观察以匹配策略期望的内容，因为命名键被编码在归一化统计层内。
  这将在即将推出的 Pipeline PR 中修复。

- **动作**
  - `Box(-1, 1, shape=(7,))` 空间中的连续控制值。

## 使用 LIBERO 进行训练

在 LIBERO 任务上训练时，确保你的数据集 parquet 和元数据键遵循 LeRobot 约定。

环境期望：

- `observation.state` → 8 维 agent 状态
- `observation.images.image` → 主相机（`agentview_image`）
- `observation.images.image2` → 手腕相机（`robot0_eye_in_hand_image`）

⚠️ 提前清理数据集**更干净、更高效**，而不是在代码内部重新映射键。
为了避免潜在的不匹配和键错误，我们提供了一个**预处理的 LIBERO 数据集**，它与当前的 LeRobot 代码库完全兼容，不需要额外的操作：
👉 [HuggingFaceVLA/libero](https://huggingface.co/datasets/HuggingFaceVLA/libero)

作为参考，这是 Physical Intelligence 发布的**原始数据集**：
👉 [physical-intelligence/libero](https://huggingface.co/datasets/physical-intelligence/libero)

---

### 示例训练命令

```bash
lerobot-train \
  --policy.type=smolvla \
  --policy.repo_id=${HF_USER}/libero-test \
  --policy.load_vlm_weights=true \
  --dataset.repo_id=HuggingFaceVLA/libero \
  --env.type=libero \
  --env.task=libero_10 \
  --output_dir=./outputs/ \
  --steps=100000 \
  --batch_size=4 \
  --eval.batch_size=1 \
  --eval.n_episodes=1 \
  --eval_freq=1000 \
```

---

### 关于渲染的说明

LeRobot 使用 MuJoCo 进行仿真。你需要在训练或评估之前设置渲染后端：

- `export MUJOCO_GL=egl` → 用于无头服务器（例如 HPC、云）

## 重现 π₀.₅ 结果

我们使用 LeRobot 实现在 LIBERO 基准测试上重现 π₀.₅ 的结果。我们采用 Physical Intelligence LIBERO 基础模型（`pi05_libero`），在 8 个 H100 GPU 上使用 [HuggingFace LIBERO 数据集](https://huggingface.co/datasets/HuggingFaceVLA/libero)，以 bfloat16 格式、batch size 为 256 进行额外 6k 步的微调。

微调后的模型可以在这里找到：

- **π₀.₅ LIBERO**：[lerobot/pi05_libero_finetuned](https://huggingface.co/lerobot/pi05_libero_finetuned)

然后，我们通过运行以下命令，使用 LeRobot LIBERO 实现评估微调后的模型：

```bash
lerobot-eval \
  --output_dir=/logs/ \
  --env.type=libero \
  --env.task=libero_spatial,libero_object,libero_goal,libero_10 \
  --eval.batch_size=1 \
  --eval.n_episodes=10 \
  --policy.path=pi05_libero_finetuned \
  --policy.n_action_steps=10 \
  --output_dir=./eval_logs/ \
  --env.max_parallel_tasks=1
```

**注意：** 我们设置 `n_action_steps=10`，与原始 OpenPI 实现类似。

### 结果

我们在 LIBERO 基准测试上获得了以下结果：

| Model    | LIBERO Spatial | LIBERO Object | LIBERO Goal | LIBERO 10 | Average  |
| -------- | -------------- | ------------- | ----------- | --------- | -------- |
| **π₀.₅** | 97.0           | 99.0          | 98.0        | 96.0      | **97.5** |

| **π₀.₅** | 98.8           | 98.2          | 98.0        | 92.4      | **96.85** |
