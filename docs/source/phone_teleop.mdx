# 手机遥操作（Phone Teleoperation）

使用您的手机（iOS 或 Android）来控制您的机器人。

**在本指南中您将学习：**

- 如何连接 iOS/Android 手机
- 手机姿态如何映射到机器人末端执行器（End-Effector, EE）目标
- 如何调整安全限制、夹爪控制和逆运动学（IK）设置

要使用手机控制机器人，请安装相关依赖：

```bash
pip install lerobot[phone]
```

## 开始使用

### 支持的平台

- iOS：使用 HEBI Mobile I/O 应用（ARKit 姿态 + 按钮）。首先下载该应用并打开它，示例程序将在您的网络上发现它并流式传输手机姿态和输入。
- Android：使用 `teleop` 包（WebXR）。当您启动 Python 进程时，它会打印一个本地 URL。在手机上打开该链接，点击 Start，然后使用 Move 来流式传输姿态。

链接：

- Android WebXR 库：[PyPI 上的 `teleop`](https://pypi.org/project/teleop/)
- iOS 应用：[HEBI Mobile I/O](https://docs.hebi.us/tools.html#mobile-io)

### 手机方向和控制

- 方向：握持手机时屏幕朝上，顶部边缘指向与机器人夹爪相同的方向。这确保校准将手机坐标系与机器人坐标系对齐，使运动感觉自然，参见下图。
- 启用/禁用：
  - iOS：按住 `B1` 启用遥操作，松开停止。第一次按下会捕获参考姿态。
  - Android：按住 `Move` 按钮，松开停止。第一次按下会捕获参考姿态。
- 夹爪控制：
  - iOS：模拟输入 `A3` 控制夹爪速度输入。
  - Android：按钮 `A` 和 `B` 充当增量/减量（A 打开，B 关闭）。您可以在 `GripperVelocityToJoint` 步骤中调整速度。

### 步骤 1：选择平台

修改示例以在 `PhoneConfig` 中使用 `PhoneOS.IOS` 或 `PhoneOS.ANDROID`。API 在各平台上完全相同，只有输入源不同。所有示例都在 `examples/` 目录下，有 `phone_so100_*.py` 变体。

遥操作示例：

```python
from lerobot.teleoperators.phone.config_phone import PhoneConfig, PhoneOS

teleop_config = PhoneConfig(phone_os=PhoneOS.IOS)  # 或 PhoneOS.ANDROID
teleop_device = Phone(teleop_config)
```

### 步骤 2：连接并校准

当创建 `Phone(teleop_config)` 并调用 `connect()` 时，会自动提示校准。将手机保持在上述方向，然后：

- iOS：按住 `B1` 捕获参考姿态。
- Android：在 WebXR 页面上按 `Move` 按钮捕获参考姿态。

为什么要校准？我们捕获当前姿态，以便后续姿态在机器人对齐的坐标系中表达。当您再次按下按钮启用控制时，位置会被重新捕获，以避免手机在禁用期间重新定位时的漂移。

### 步骤 3：运行示例

运行示例脚本进行遥操作、记录数据集、重放数据集或评估策略。

所有脚本都假设您已配置好机器人（例如 SO-100 从动臂）并设置了正确的串口。

此外，您需要**将机器人的 urdf 复制到示例文件夹**。对于本教程中的示例（使用 SO100/SO101），强烈建议使用 [SO-ARM100 仓库](https://github.com/TheRobotStudio/SO-ARM100/blob/main/Simulation/SO101/so101_new_calib.urdf)中的 urdf。

- 运行此示例进行遥操作：

  ```bash
  python examples/phone_to_so100/teleoperate.py
  ```

运行示例后：

- Android：启动脚本后，在手机上打开打印的本地 URL，点击 Start，然后按住 Move。
- iOS：首先打开 HEBI Mobile I/O；B1 启用运动，A3 控制夹爪。

此外，您可以通过编辑示例中显示的处理器步骤来自定义映射或安全限制。您还可以重新映射输入（例如使用不同的模拟输入）或通过修改输入和运动学步骤将流水线适配到其他机器人（例如 LeKiwi）。更多信息请参见[机器人和遥操作器的处理器](./processors_robots_teleop)指南。

- 运行此示例记录数据集，它保存绝对末端执行器观测和动作：

  ```bash
  python examples/phone_to_so100/record.py
  ```

- 运行此示例重放记录的片段：

  ```bash
  python examples/phone_to_so100/replay.py
  ```

- 运行此示例评估预训练策略：

  ```bash
  python examples/phone_to_so100/evaluate.py
  ```

### 重要的流水线步骤和选项

- 运动学在多个步骤中使用。我们使用 [Placo](https://github.com/Rhoban/placo)，它是 Pinocchio 的包装器，用于处理我们的运动学。我们通过传递机器人的 URDF 和目标坐标系来构造运动学对象。我们将 `target_frame_name` 设置为夹爪坐标系。

  ```python
  kinematics_solver = RobotKinematics(
    urdf_path="./SO101/so101_new_calib.urdf",
    target_frame_name="gripper_frame_link",
    joint_names=list(robot.bus.motors.keys()),
  )
  ```

- `MapPhoneActionToRobotAction` 步骤将校准的手机姿态和输入转换为目标增量和夹爪命令，下面显示了该步骤的输出。

  ```python
  action["enabled"] = enabled
        action["target_x"] = -pos[1] if enabled else 0.0
        action["target_y"] = pos[0] if enabled else 0.0
        action["target_z"] = pos[2] if enabled else 0.0
        action["target_wx"] = rotvec[1] if enabled else 0.0
        action["target_wy"] = rotvec[0] if enabled else 0.0
        action["target_wz"] = -rotvec[2] if enabled else 0.0
        action["gripper_vel"] = gripper_vel  # 禁用时仍发送夹爪动作
  ```

- `EEReferenceAndDelta` 步骤将目标增量转换为绝对期望末端执行器姿态，在启用时存储参考。`end_effector_step_sizes` 是末端执行器姿态的步长大小，可以修改以改变运动速度。

  ```python
  EEReferenceAndDelta(
      kinematics=kinematics_solver,
      end_effector_step_sizes={"x": 0.5, "y": 0.5, "z": 0.5},
      motor_names=list(robot.bus.motors.keys()),
      use_latched_reference=True,
  ),
  ```

- `EEBoundsAndSafety` 步骤将末端执行器运动限制在工作空间内，并检查大的末端执行器步进跳变以确保安全。`end_effector_bounds` 是末端执行器姿态的边界，可以修改以改变工作空间。`max_ee_step_m` 是末端执行器姿态的步长限制，可以修改以改变安全限制。

  ```python
  EEBoundsAndSafety(
      end_effector_bounds={"min": [-1.0, -1.0, -1.0], "max": [1.0, 1.0, 1.0]},
      max_ee_step_m=0.10,
  )
  ```

- `GripperVelocityToJoint` 步骤使用当前测量状态将类似速度的夹爪输入转换为绝对夹爪位置。`speed_factor` 是速度乘以的因子。

  ```python
  GripperVelocityToJoint(speed_factor=20.0)
  ```

#### 不同的逆运动学初始猜测

我们在运动学步骤中使用不同的逆运动学初始猜测。作为初始猜测，使用当前测量的关节或前一个逆运动学解。

- 闭环（在 record/eval 中使用）：设置 `initial_guess_current_joints=True`，因此逆运动学每帧从测量的关节开始。

  ```python
  InverseKinematicsEEToJoints(
      kinematics=kinematics_solver,
      motor_names=list(robot.bus.motors.keys()),
      initial_guess_current_joints=True,  # 闭环
  )
  ```

- 开环（在 replay 中使用）：设置 `initial_guess_current_joints=False`，因此逆运动学从前一个逆运动学解继续，而不是从测量状态。这在我们无反馈重放时保持动作稳定性。

  ```python
  InverseKinematicsEEToJoints(
      kinematics=kinematics_solver,
      motor_names=list(robot.bus.motors.keys()),
      initial_guess_current_joints=False,  # 开环
  )
  ```

### 流水线步骤解释

- MapPhoneActionToRobotAction：将校准的手机姿态和输入转换为目标增量和夹爪命令。运动由启用信号控制（iOS 上的 B1，Android 上的 Move）。
- EEReferenceAndDelta：在启用时锁存参考末端执行器姿态，并将其与目标增量组合以在每帧产生绝对期望末端执行器姿态。禁用时，它继续发送最后命令的姿态。
- EEBoundsAndSafety：将末端执行器姿态限制在工作空间内，并对跳变进行速率限制以确保安全。还声明 `action.ee.*` 特征。
- InverseKinematicsEEToJoints：通过逆运动学将末端执行器姿态转换为关节位置。对于闭环控制，建议使用 `initial_guess_current_joints=True`；对于开环重放，设置为 `False` 以保持稳定性。
- GripperVelocityToJoint：使用当前测量状态将类似速度的夹爪输入积分为绝对夹爪位置。
- ForwardKinematicsJointsToEE：从观测到的关节计算 `observation.state.ee.*`，用于记录和训练末端执行器状态。

### 故障排除

- iOS 未被发现：确保 HEBI Mobile I/O 已打开，并且您的笔记本电脑/手机在同一网络上。
- Android URL 无法访问：检查您使用的是 `https` 而不是 `http`，使用脚本打印的确切 IP，并允许浏览器进入并忽略证书问题。
- 运动感觉反向：调整 `MapPhoneActionToRobotAction` 中的符号翻转或交换轴以匹配您的设置。
