# Reachy 2

Reachy 2 是由 Pollen Robotics 制造的开源人形机器人，专为具身 AI 和真实世界应用的开发而设计。
查看 [Pollen Robotics 网站](https://www.pollen-robotics.com/reachy/)，或访问 [Reachy 2 文档](https://docs.pollen-robotics.com/) 了解有关该平台的更多信息！

## 遥操作 Reachy 2

目前，有两种方式可以遥操作 Reachy 2：

- Pollen Robotics 的 VR 遥操作（不包含在 LeRobot 中）。
- 机器人到机器人遥操作（使用一个 Reachy 2 控制另一个）。

## Reachy 2 仿真

**（仅限 Linux）** 您可以使用提供的 [Docker 镜像](https://hub.docker.com/r/pollenrobotics/reachy2_core) 在仿真中运行 Reachy 2（Gazebo 或 MuJoCo）。

1. 安装 [Docker Engine](https://docs.docker.com/engine/)。
2. 运行（对于 MuJoCo）：

```
docker run --rm -it \
  --name reachy \
  --privileged \
  --network host \
  --ipc host \
  --device-cgroup-rule='c 189:* rwm' \
  --group-add audio \
  -e ROS_DOMAIN_ID="$ROS_DOMAIN_ID" \
  -e DISPLAY="$DISPLAY" \
  -e RCUTILS_CONSOLE_OUTPUT_FORMAT="[{severity}]: {message}" \
  -e REACHY2_CORE_SERVICE_FAKE="${REACHY2_CORE_SERVICE_FAKE:-true}" \
  -v /dev:/dev \
  -v "$HOME/.reachy_config":/home/reachy/.reachy_config_override \
  -v "$HOME/.reachy.log":/home/reachy/.ros/log \
  -v /usr/lib/x86_64-linux-gnu:/opt/host-libs \
  --entrypoint /package/launch.sh \
  pollenrobotics/reachy2_core:1.7.5.9_deploy \
  start_rviz:=true start_sdk_server:=true mujoco:=true
```

> 如果 MuJoCo 运行缓慢（仿真频率低），请在上一个命令中追加 `-e LD_LIBRARY_PATH="/opt/host-libs:$LD_LIBRARY_PATH" \` 以提高性能：

## 设置

### 前提条件

- 在您的机器人上，检查**服务镜像**是否满足最低版本要求：
  - **reachy2-core >= 1.7.5.2**
  - **webrtc >= 2.0.1.1**

然后，如果您想使用 VR 遥操作：

- 安装 [Reachy 2 遥操作应用程序](https://docs.pollen-robotics.com/teleoperation/teleoperation-introduction/discover-teleoperation/)。
  使用版本 **>=v1.2.0**

我们建议使用两台计算机：一台用于遥操作（需要 Windows），另一台用于使用 LeRobot 进行记录。

### 安装 LeRobot

遵循 [安装说明](https://github.com/huggingface/lerobot#installation) 来安装 LeRobot。

安装带有 Reachy 2 依赖项的 LeRobot：

```bash
pip install -e ".[reachy2]"
```

### （可选但推荐）安装 pollen_data_acquisition_server

您如何管理 Reachy 2 记录会话由您决定，但**最简单**的方法是使用此服务器，这样您就可以直接从 VR 遥操作应用程序控制会话。

> **注意：** 目前，只有 VR 遥操作应用程序可以作为此服务器的客户端，因此此步骤主要针对遥操作。您可以自由开发自定义客户端来根据您的需求管理会话。

在您的 LeRobot 环境中，从源代码安装服务器：

```bash
git clone https://github.com/pollen-robotics/pollen_data_acquisition_server.git
cd pollen_data_acquisition_server
pip install -e .
```

在 [这里找到 pollen_data_acquisition_server 文档](https://github.com/pollen-robotics/pollen_data_acquisition_server)。

## 步骤 1：记录

### 获取 Reachy 2 IP 地址

在开始遥操作和数据记录之前，请找到 [机器人的 IP 地址](https://docs.pollen-robotics.com/getting-started/setup-reachy2/connect-reachy2/)。
我们强烈建议通过**以太网**连接所有设备（PC 和机器人）。

### 启动记录

使用 Reachy 2 VR 遥操作应用程序时，有两种管理记录会话的方式：

- **使用数据采集服务器（推荐用于 VR 遥操作）**：VR 应用程序编排会话（通过服务器告诉 LeRobot 何时创建数据集、启动/停止回合），同时还控制机器人的运动。
- **使用 LeRobot 的记录脚本**：LeRobot 拥有会话控制权并决定何时启动/停止回合。如果您还使用 VR 遥操作应用程序，它仅用于运动控制。

### 选项 1：使用 Pollen 数据采集服务器（推荐用于 VR 遥操作）

确保您已按设置部分中的说明安装了 pollen_data_acquisition_server。

启动数据采集服务器，以便能够直接从遥操作应用程序管理您的会话：

```bash
python -m pollen_data_acquisition_server.server
```

然后进入遥操作应用程序并选择"Data acquisition session"。
您最后可以按照显示的屏幕设置您的会话。

> 即使没有 VR 应用程序，您也可以使用自己的客户端实现来使用 `pollen_data_acquisition_server`。

### 选项 2：使用 lerobot.record

LeRobot 的记录功能完全支持 Reachy 2。
如果您选择此选项但仍想使用 VR 遥操作应用程序，请在应用程序中选择"Standard session"。

**示例：在不使用移动底座的情况下启动记录：**
首先将 reachy2 和 reachy2_teleoperator 添加到记录脚本的导入中。然后您可以使用以下命令：

```bash
python -m lerobot.record \
    --robot.type=reachy2 \
    --robot.ip_address=192.168.0.200 \
    --robot.id=r2-0000 \
    --robot.use_external_commands=true \
    --robot.with_mobile_base=false \
    --teleop.type=reachy2_teleoperator \
    --teleop.ip_address=192.168.0.200 \
    --teleop.with_mobile_base=false \
    --dataset.repo_id=pollen_robotics/record_test \
    --dataset.single_task="Reachy 2 recording test" \
    --dataset.num_episodes=1 \
    --dataset.episode_time_s=5 \
    --dataset.fps=15 \
    --dataset.push_to_hub=true \
    --dataset.private=true \
    --display_data=true
```

#### 特定选项

**扩展设置概述（包括所有选项）：**

```bash
python -m lerobot.record \
    --robot.type=reachy2 \
    --robot.ip_address=192.168.0.200 \
    --robot.use_external_commands=true \
    --robot.with_mobile_base=true \
    --robot.with_l_arm=true \
    --robot.with_r_arm=true \
    --robot.with_neck=true \
    --robot.with_antennas=true \
    --robot.with_left_teleop_camera=true \
    --robot.with_right_teleop_camera=true \
    --robot.with_torso_camera=false \
    --robot.disable_torque_on_disconnect=false \
    --robot.max_relative_target=5.0 \
    --teleop.type=reachy2_teleoperator \
    --teleop.ip_address=192.168.0.200 \
    --teleop.use_present_position=false \
    --teleop.with_mobile_base=false \
    --teleop.with_l_arm=true \
    --teleop.with_r_arm=true \
    --teleop.with_neck=true \
    --teleop.with_antennas=true \
    --dataset.repo_id=pollen_robotics/record_test \
    --dataset.single_task="Reachy 2 recording test" \
    --dataset.num_episodes=1 \
    --dataset.episode_time_s=5 \
    --dataset.fps=15 \
    --dataset.push_to_hub=true \
    --dataset.private=true \
    --display_data=true
```

##### `--robot.use_external_commands`

确定 LeRobot robot.send_action() 是否向机器人发送命令。
在使用 VR 遥操作应用程序时**必须**设置为 false，因为应用程序已经发送命令。

##### `--teleop.use_present_position`

确定遥操作器读取机器人的目标位置还是当前位置。
如果使用一个柔顺的 Reachy 2 来控制另一个，则必须设置为 true。

##### 使用相关部分

根据我们的初步测试，当某些策略只移动部分关节时，记录**所有**关节可能会降低模型质量。
为避免这种情况，您可以使用以下方式从记录和回放中排除特定部分：

````
--robot.with_<part>=false
```,
其中 `<part>` 为以下之一：`mobile_base`、`l_arm`、`r_arm"`、`neck`、`antennas`。
它确定相应的部分是否记录在观察中。如果未设置，则为 True。

默认情况下，**记录所有部分**。

`reachy2_teleoperator` 中也提供了相同的按部分机制。

````
--teleop.with_<part>
```
其中 `<part>` 为以下之一：`mobile_base`、`l_arm`、`r_arm"`、`neck`、`antennas`。
确定相应的部分是否记录在动作中。如果未设置，则为 True。

> **重要：** 在给定的会话中，机器人和遥操作器上的**启用部分必须匹配**。
例如，如果机器人使用 `--robot.with_mobile_base=false` 运行，则遥操作器必须禁用相同的部分 `--teleoperator.with_mobile_base=false`。

##### 使用相关摄像头

您可以对**摄像头**执行相同的操作。默认情况下，仅记录**遥操作摄像头**（`left_teleop_camera` 和 `right_teleop_camera`）。使用以下方式启用或禁用每个摄像头：

```

--robot.with_left_teleop_camera=<true|false>
--robot.with_right_teleop_camera=<true|false>
--robot.with_torso_camera=<true|false>

````


## 步骤 2：回放

确保机器人配置的部分与数据集相同：

```bash
python -m lerobot.replay \
    --robot.type=reachy2 \
    --robot.ip_address=192.168.0.200 \
    --robot.use_external_commands=false \
    --robot.with_mobile_base=false \
    --dataset.repo_id=pollen_robotics/record_test \
    --dataset.episode=0
    --display_data=true
````

## 步骤 3：训练

```bash
python -m lerobot.scripts.train \
  --dataset.repo_id=pollen_robotics/record_test \
  --policy.type=act \
  --output_dir=outputs/train/reachy2_test \
  --job_name=reachy2 \
  --policy.device=mps \
  --wandb.enable=true \
  --policy.repo_id=pollen_robotics/record_test_policy
```

## 步骤 4：评估

```bash
python -m lerobot.record \
  --robot.type=reachy2 \
  --robot.ip_address=192.168.0.200 \
  --display_data=false \
  --dataset.repo_id=pollen_robotics/eval_record_test \
  --dataset.single_task="Evaluate reachy2 policy" \
  --dataset.num_episodes=10 \
  --policy.path=outputs/train/reachy2_test/checkpoints/last/pretrained_model
```
