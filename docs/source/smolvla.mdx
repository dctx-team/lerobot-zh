# SmolVLA

SmolVLA 是 Hugging Face 推出的轻量级机器人基础模型（foundation model）。它专为在 LeRobot 数据集上进行简易微调而设计，可以帮助加速您的开发！

## 设置您的环境

1. 按照我们的[安装指南](./installation)安装 LeRobot。
2. 通过运行以下命令安装 SmolVLA 依赖项：

   ```bash
   pip install -e ".[smolvla]"
   ```

## 收集数据集

SmolVLA 是一个基础模型，因此需要在您自己的数据上进行微调，才能在您的设置中获得最佳性能。
我们建议首先记录您任务的约 50 个片段（episodes）。请遵循我们的指南开始：[记录数据集](./il_robots)

<Tip>

在您的数据集中，请确保为您引入的每个变化（例如，如果是立方体拾取放置任务，则为桌子上的立方体位置）提供足够的演示。

我们建议查看下面链接的数据集作为参考，该数据集在 [SmolVLA 论文](https://huggingface.co/papers/2506.01844)中使用：

🔗 [SVLA SO100 PickPlace](https://huggingface.co/spaces/lerobot/visualize_dataset?path=%2Flerobot%2Fsvla_so100_pickplace%2Fepisode_0)

在这个数据集中，我们在 5 个不同的立方体位置记录了 50 个片段。对于每个位置，我们收集了 10 个拾取和放置交互的片段。这种结构，多次重复每个变化，帮助模型更好地泛化。我们尝试过包含 25 个片段的类似数据集，但数量不足导致性能不佳。因此，数据质量和数量绝对是关键。
在您的数据集在 Hub 上可用后，您就可以使用我们的微调脚本将 SmolVLA 适配到您的应用了。

</Tip>

## 在您的数据上微调 SmolVLA

使用 [`smolvla_base`](https://hf.co/lerobot/smolvla_base)，我们预训练的 450M 模型，并在您的数据上对其进行微调。
在单个 A100 GPU 上训练模型 20k 步大约需要 4 小时。您应该根据性能和您的用例调整步数。

如果您没有 GPU 设备，可以使用我们在 [![Google Colab](https://colab.research.google.com/assets/colab-badge.svg)](https://colab.research.google.com/github/huggingface/notebooks/blob/main/lerobot/training-smolvla.ipynb) 上的笔记本进行训练

使用 `--dataset.repo_id` 将您的数据集传递给训练脚本。如果您想测试您的安装，请运行以下命令，我们使用为 [SmolVLA 论文](https://huggingface.co/papers/2506.01844)收集的数据集之一。

```bash
cd lerobot && lerobot-train \
  --policy.path=lerobot/smolvla_base \
  --dataset.repo_id=${HF_USER}/mydataset \
  --batch_size=64 \
  --steps=20000 \
  --output_dir=outputs/train/my_smolvla \
  --job_name=my_smolvla_training \
  --policy.device=cuda \
  --wandb.enable=true
```

<Tip>
  您可以从较小的批处理大小（batch size）开始，如果 GPU 允许，可以逐步增加，只要加载时间保持较短即可。
</Tip>

微调是一门艺术。要全面了解微调的选项，请运行

```bash
lerobot-train --help
```

## 评估微调模型并实时运行

与记录片段时类似，建议您登录到 HuggingFace Hub。您可以按照相应的步骤操作：[记录数据集](./il_robots)。
登录后，您可以通过以下方式在您的设置中运行推理：

```bash
lerobot-record \
  --robot.type=so101_follower \
  --robot.port=/dev/ttyACM0 \ # <- 使用您的端口
  --robot.id=my_blue_follower_arm \ # <- 使用您的机器人 ID
  --robot.cameras="{ front: {type: opencv, index_or_path: 8, width: 640, height: 480, fps: 30}}" \ # <- 使用您的摄像头
  --dataset.single_task="Grasp a lego block and put it in the bin." \ # <- 使用您在数据集记录中使用的相同任务描述
  --dataset.repo_id=${HF_USER}/eval_DATASET_NAME_test \  # <- 这将是 HF Hub 上的数据集名称
  --dataset.episode_time_s=50 \
  --dataset.num_episodes=10 \
  # <- 如果您想在片段之间进行远程操作，可选的 Teleop \
  # --teleop.type=so100_leader \
  # --teleop.port=/dev/ttyACM0 \
  # --teleop.id=my_red_leader_arm \
  --policy.path=HF_USER/FINETUNE_MODEL_NAME # <- 使用您的微调模型
```

根据您的评估设置，您可以配置要为评估套件记录的持续时间和片段数量。
